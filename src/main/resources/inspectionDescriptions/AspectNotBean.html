Reports @Aspect classes that are not registered as Spring Beans.

For Spring AOP to apply the aspect, the class must be a Spring-managed bean (annotated with @Component, @Service, @Repository, etc., including custom meta-annotations built on top of those stereotypes). Without a bean annotation, Spring will not detect or apply the aspect at runtime.

Quick fix: Adds @Component only when no Spring bean stereotype is present.

  • Idempotent: does not add duplicate stereotype annotations.
  • Safe: does not modify the class when it is already recognized as a bean.
  • Non-destructive: only appends the missing stereotype annotation.

Note: This warning does not apply to standalone AspectJ (compile-time weaving).
